# 场景基础视频处理实现总结

## 目标

通过以下方式将VideoR从基于帧的视频处理转换为基于场景的处理：
1. 使用对角帧差分矩阵自动检测视频场景边界
2. 组织围绕检测到的场景的训练数据和计算
3. 在SFT和RL训练中实现更高效的计算资源使用

## 已完成的实现

### 1. 核心场景检测模块 ✓

**文件**: `RL/my_qwen_vl_utils/scene_detection.py`

**功能**:
- `compute_frame_difference_matrix(video_tensor, method='l2')` - 计算相邻帧之间的L2/L1距离
- `detect_scene_boundaries(video_tensor, percentile=90, min_scene_length=1, method='l2')` - 基于自适应阈值的边界检测
- `group_frames_by_scene(video_tensor, boundaries)` - 将帧分组为场景间隔
- `get_scene_statistics(video_tensor, scenes)` - 计算场景统计信息
- `map_raw_scenes_to_processed(raw_scenes, raw_fps, processed_fps, total_processed_frames)` - 在原始/处理视频之间映射边界

**关键特性**:
- 使用帧差分的90th百分位作为自适应阈值
- 自动适应视频内容
- 处理重新采样期间的帧计数变化
- 内存开销最小

### 2. 增强的视觉处理 ✓

## 算法细节

### 场景检测过程

1. **帧表示**
   - 将帧转换为浮点数（0-1范围）
   - 展平空间维度：(C, H, W) → (C*H*W)

2. **差分计算**
   - 计算相邻帧向量之间的L2范数
   - 结果：T帧视频的长度为 (T-1) 的1D数组

3. **自适应阈值处理**
   - 计算差分分布的90th百分位
   - 阈值自动适应视频内容统计信息

4. **边界检测**
   - 差异 > 阈值的帧索引 = 场景边界
   - 最少强制1帧场景
   - 帧0始终包含

5. **场景分组**
   - 将边界转换为连续的 (start, end) 间隔
   - 非重叠覆盖整个视频

### 边界映射

当视频经过重新采样时：
- 将帧索引转换为时间：`time = frame_idx / fps`
- 在新fps处重新计算帧索引：`new_idx = time * new_fps`
- 确保跨处理管道的时间一致性

## 数据流

### RL训练

```
视频文件
    ↓
fetch_video_raw_with_scenes()
    ├─ 加载原始视频 (T_raw, C, H, W)
    ├─ 检测边界（原始帧索引）
    └─ 返回: (video, fps, scenes)
    ↓
resample_video_from_raw()
    ├─ 调整帧大小
    └─ 调整帧计数
    ↓
map_raw_scenes_to_processed()
    └─ 映射到处理后的帧索引
    ↓
MultiModalDatasetVideo.__getitem__()
    ├─ 存储处理后的视频 (T_proc, C, H', W')
    ├─ 存储 scene_boundaries: [(s1, e1), (s2, e2), ...]
    └─ 在批次中返回
    ↓
RL训练器
    ├─ 通过索引按场景访问帧
    ├─ 可选：计算每个场景的奖励
    └─ 可选：场景感知注意力掩码
```

### SFT训练

```
视频文件
    ↓
_regularize_videos_av/decord() [如果 USE_SCENE_DETECTION=true]
    ├─ 加载帧
    ├─ 检测场景边界
    └─ 返回: videos + scene_boundaries
    ↓
SFT数据集
    └─ 如果需要，按场景组织帧
    ↓
SFT训练器
    └─ 用于训练组织的场景信息
```
