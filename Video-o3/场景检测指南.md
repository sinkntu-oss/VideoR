# 场景基础视频处理实现指南

## 概述

本文档描述了VideoR项目的场景基础视频处理的实现。该系统使用对角帧差分矩阵自动检测视频场景边界，实现更智能的帧采样和训练组织。

## 更改摘要

### 新文件

1. **`RL/my_qwen_vl_utils/scene_detection.py`**
   - 核心场景检测模块
   - 关键函数：
     - `compute_frame_difference_matrix()` - 计算相邻帧之间的差异

## 场景检测细节

### 算法

1. **帧差分计算**
   - 计算相邻帧表示之间的L2距离
   - 公式：`diff[t] = ||frame[t+1] - frame[t]||_2`
   - 帧转换为浮点数并展平（C*H*W）以提高效率

2. **自适应阈值**
   - 阈值计算为差分分布的百分位数（默认：90th百分位）
   - 百分位允许自动适应视频内容
   - 对实际内容变化敏感，而不是绝对值

3. **边界检测**
   - 差异 > 阈值的帧索引成为场景边界
   - 强制执行最小场景长度（默认：1帧）
   - 帧0总是包含为第一个场景的开始

4. **场景分组**
   - 连续的边界分组为 (start_frame, end_frame) 间隔
   - 非重叠且连续覆盖整个视频

### 帧索引映射

当视频经过重新采样（原始→处理）时：

```
原始视频：300帧 @ 30fps
  场景1：帧0-100    (0.0s - 3.33s)
  场景2：帧100-200  (3.33s - 6.67s)
  场景3：帧200-300  (6.67s - 10.0s)

处理后视频：20帧 @ 2fps (总持续时间10s)
  场景1：帧0-7      (0.0s - 3.5s)
  场景2：帧7-13     (3.5s - 6.5s)
  场景3：帧13-20    (6.5s - 10.0s)
```

## 数据流

### RL训练管道

```
输入视频文件
    ↓
fetch_video_raw_with_scenes()
    ├─ 加载原始视频
    └─ 检测场景边界（原始帧索引）
    ↓
resample_video_from_raw()
    ├─ 调整帧大小
    └─ 调整帧数
    ↓
map_raw_scenes_to_processed()
    └─ 将边界映射到处理后的帧索引
    ↓
MultiModalDatasetVideo
    ├─ 存储处理后的视频张量
    ├─ 存储场景边界
    └─ 在批次中输出，带 'scene_boundaries' 键
    ↓
RL训练器
    ├─ 按场景访问帧
    └─ 计算每个场景的奖励
```

### SFT训练管道

```
输入视频文件
    ↓
_regularize_videos_av() 或 _regularize_videos_decord()
    ├─ 加载帧
    ├─ 检测场景边界（如果 USE_SCENE_DETECTION=true）
    └─ 在输出中返回 'scene_boundaries'
    ↓
SFT数据集
    └─ 用于帧组织
    ↓
SFT训练器
```

## 与奖励模型集成

场景信息可被奖励模型用于：

1. **计算每个场景的指标**
   - 每个检测到的场景的准确率/损失
   - 识别有问题的场景

2. **场景感知注意力**
   - 在场景内屏蔽注意力
   - 强调场景边界

3. **时间锚定**
   - 验证模型输出与场景转换对齐
   - 使用场景进行时间一致性检查

**示例：**

```python
def reward_fn(scene_boundaries, model_output, ground_truth):
    """计算按场景分组的奖励。"""
    rewards = []
    for (start, end) in scene_boundaries:
        scene_pred = model_output[start:end]
        scene_truth = ground_truth[start:end]
        scene_reward = compute_accuracy(scene_pred, scene_truth)
        rewards.append(scene_reward)
    return np.mean(rewards)
```

## 性能考虑

### 计算成本

- **场景检测开销**：~5-10% 每个视频（一次性，在加载期间）
- **内存**：最少（存储 (int, int) 元组列表）
- **训练影响**：无（检测在预处理期间进行）

### 优化机会

1. **批处理**：并行检测多个视频的场景
2. **缓存**：缓存重复视频的检测到的边界
3. **自适应百分位**：根据视频内容类型调整百分位

## 故障排除

### 场景检测不工作

**症状**：`scene_boundaries` 不在输出中

**解决方案**：
- RL：检查数据集构造函数中 `use_scene_detection=True`
- SFT：检查 `USE_SCENE_DETECTION=true` 环境变量已设置
- 检查日志中的导入错误

### 检测到太多/太少场景

**症状**：场景边界似乎不正确

**解决方案**：
- 调整百分位：较低值 → 更多边界
- 检查视频内容：均匀帧 → 更少边界

### 内存问题

**症状**：场景检测期间内存不足

**解决方案**：
- 场景检测内存高效；检查视频加载
- 对非常长的视频考虑 `detect_scenes=False`
- 在较小的批次中启用
