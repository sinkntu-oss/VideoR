## 幕后工作原理

1. **在数据加载期间**：
   - 视频逐帧加载
   - 比较相邻帧（L2距离）
   - 在差异峰值处检测场景边界
   - 存储边界为帧索引

2. **在训练中**：
   - 场景边界在批次中作为 `scene_boundaries` 键可用
   - 您可以用它们来：
     - 计算每个场景的奖励
     - 场景感知的损失加权
     - 注意力掩码
     - 课程学习

3. **对训练速度没有影响**：
   - 检测在加载期间进行一次
   - 最小开销（~5-10%）
   - 对训练循环没有影响

## 示例：每个场景的奖励

```python
def compute_scene_rewards(predictions, ground_truth, scene_boundaries):
    """为每个场景分别计算奖励。"""
    
    all_rewards = []
    
    for scene_start, scene_end in scene_boundaries:
        scene_pred = predictions[scene_start:scene_end]
        scene_truth = ground_truth[scene_start:scene_end]
        
        # 计算此场景的准确率
        correct = (scene_pred.argmax(-1) == scene_truth).float()
        scene_reward = correct.mean()
        
        all_rewards.append(scene_reward)
    
    return torch.mean(torch.tensor(all_rewards))


# 在训练循环中
batch = next(iter(dataloader))
rewards = compute_scene_rewards(
    batch['predictions'],
    batch['ground_truth'],
    batch['scene_boundaries']
)
```

## 调整检测敏感度

如果检测找到的场景太多或太少：

**在RL中（vision_process.py）**：
```python

# 不敏感（更少场景）

```

**在SFT中（mm_plugin.py）**：
修改源代码中调用场景检测的百分位数参数。

## 故障排除

| 问题 | 解决方案 |
|------|--------|
| 场景边界不在输出中（RL） | 检查配置或数据集初始化中是否有 `use_scene_detection=True` |
| 场景边界不在输出中（SFT） | 检查是否设置了 `USE_SCENE_DETECTION=true` 环境变量 |
| 训练更慢 | 场景检测仅在预处理期间进行；检查总数据大小 |
| 场景过多或过少 | 调整百分位参数（85=更敏感，95=不敏感） |
